# Getting Started

Welcome to the **new documentation generator**! This guide will help you get started with creating beautiful, multilingual documentation sites.

## Overview

The new documentation generator is a registry-driven system that allows you to:

- 🚀 **Fast**: Built with Astro for optimal performance
- 🌍 **Multilingual**: Support for 15+ languages out of the box
- 🔍 **Searchable**: Powered by Pagefind for instant search
- 📦 **Versioned**: Multiple version support with easy switching
- 🎨 **Customizable**: Theming system with CSS variables
- ♿ **Accessible**: WCAG 2.1 AA compliant

## Features

### Registry-Driven Architecture

All documentation structure is defined in a JSON registry file. This provides:

- **Single source of truth** for project structure
- **Type-safe** configuration with JSON schema validation
- **Version control** friendly format
- **Easy migration** from existing documentation

### Shared Packages

The generator is built with reusable packages:

| Package | Description |
|---------|-------------|
| `@docs/generator` | Registry loading, route generation, sidebar/sitemap creation |
| `@docs/ui` | 30+ Astro UI components (Navigation, Sidebar, Cards, etc.) |
| `@docs/theme` | Design tokens, CSS variables, color palettes |
| `@docs/i18n` | Internationalization utilities for 15 languages |
| `@docs/versioning` | Version selector and diff comparison components |

### Search Functionality

Advanced search features include:

- **Faceted search**: Filter by project, version, and language
- **Pagination**: 10 results per page with navigation
- **Highlighting**: Search terms highlighted in results
- **Fast indexing**: Powered by Pagefind

## Quick Start

### 1. Installation

```bash
# Install dependencies
pnpm install @docs/generator @docs/ui @docs/theme @docs/i18n

# Or use the monorepo workspace
pnpm install
```

### 2. Create a Registry

Create a `registry/docs.json` file:

```json
{
  "$schemaVersion": "1.0.0",
  "projects": [
    {
      "id": "my-docs",
      "displayName": {
        "en": "My Documentation"
      },
      "languages": [
        {
          "code": "en",
          "displayName": "English",
          "default": true
        }
      ],
      "versions": [
        {
          "id": "v1",
          "name": "Version 1.0",
          "isLatest": true
        }
      ]
    }
  ]
}
```

### 3. Create Content

Create MDX files in `src/content/docs/`:

```
src/content/docs/
└── my-docs/
    └── v1/
        └── en/
            └── guide/
                └── getting-started.mdx
```

### 4. Build and Deploy

```bash
# Development server
pnpm dev

# Production build
pnpm build

# Preview
pnpm preview
```

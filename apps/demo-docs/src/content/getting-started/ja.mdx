# はじめに

**新しいドキュメントジェネレーター**へようこそ！このガイドでは、美しい多言語ドキュメントサイトの作成方法を説明します。

## 概要

新しいドキュメントジェネレーターは、レジストリ駆動型のシステムで、以下の機能を提供します:

- 🚀 **高速**: Astroで構築され、最適なパフォーマンスを実現
- 🌍 **多言語対応**: 15以上の言語をすぐに利用可能
- 🔍 **検索可能**: Pagefindによる瞬時の検索機能
- 📦 **バージョン管理**: 複数バージョンの簡単な切り替えをサポート
- 🎨 **カスタマイズ可能**: CSS変数によるテーマシステム
- ♿ **アクセシブル**: WCAG 2.1 AA準拠

## 主要機能

### レジストリ駆動型アーキテクチャ

すべてのドキュメント構造はJSONレジストリファイルで定義されます。これにより以下が実現します:

- プロジェクト構造の**単一信頼できる情報源**
- JSONスキーマ検証による**型安全な**設定
- **バージョン管理**に適した形式
- 既存ドキュメントからの**簡単な移行**

### 共有パッケージ

ジェネレーターは再利用可能なパッケージで構築されています:

| パッケージ | 説明 |
|---------|------|
| `@docs/generator` | レジストリ読み込み、ルート生成、サイドバー/サイトマップ作成 |
| `@docs/ui` | 30以上のAstro UIコンポーネント（ナビゲーション、サイドバー、カードなど） |
| `@docs/theme` | デザイントークン、CSS変数、カラーパレット |
| `@docs/i18n` | 15言語対応の国際化ユーティリティ |
| `@docs/versioning` | バージョンセレクターと差分比較コンポーネント |

### 検索機能

高度な検索機能には以下が含まれます:

- **ファセット検索**: プロジェクト、バージョン、言語でフィルター
- **ページネーション**: 1ページ10件の結果でナビゲーション
- **ハイライト**: 検索結果で検索語をハイライト
- **高速インデックス**: Pagefindを使用

## クイックスタート

### 1. インストール

```bash
# 依存関係のインストール
pnpm install @docs/generator @docs/ui @docs/theme @docs/i18n

# またはモノレポワークスペースを使用
pnpm install
```

### 2. レジストリの作成

`registry/docs.json`ファイルを作成:

```json
{
  "$schemaVersion": "1.0.0",
  "projects": [
    {
      "id": "my-docs",
      "displayName": {
        "ja": "私のドキュメント"
      },
      "languages": [
        {
          "code": "ja",
          "displayName": "日本語",
          "default": true
        }
      ],
      "versions": [
        {
          "id": "v1",
          "name": "バージョン 1.0",
          "isLatest": true
        }
      ]
    }
  ]
}
```

### 3. コンテンツの作成

`src/content/docs/`にMDXファイルを作成:

```
src/content/docs/
└── my-docs/
    └── v1/
        └── ja/
            └── guide/
                └── getting-started.mdx
```

### 4. ビルドとデプロイ

```bash
# 開発サーバー
pnpm dev

# プロダクションビルド
pnpm build

# プレビュー
pnpm preview
```
